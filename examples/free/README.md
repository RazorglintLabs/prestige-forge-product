# Free Tier Examples

## What this category covers

The free tier gives you proof visibility — the ability to see the current verification state of a project at a glance.

This folder will contain examples of the two main outputs of a free-tier run:

- **Badge and proof summary** — a single pass/fail verdict with proof counts, tied to a specific git commit
- **Verification history** — an ordered timeline of every check that ran, what it found, and when

These are the kinds of artifacts Prestige Forge produces at no cost.

## What you should learn from these examples

- What a badge summary looks like when all checks pass
- How the verification history is structured — chronological, per-check, with clear verdicts
- What information is included in a proof bundle without any paid features enabled
- Whether the free tier alone is useful for your needs before you consider paid tiers

## What is deliberately omitted

- **No source code.** These are output artifacts only. The engine that produces them is not included.
- **No internal file paths.** All paths and references are sanitized to the output context.
- **No implementation detail.** You will not find module names, class structures, or internal wiring here.
- **No private proof data.** When populated, examples will use Prestige Forge's own Client Zero self-verification run. No customer data will appear.
